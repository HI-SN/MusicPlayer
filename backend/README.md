1. - backend（后端部分）
     - main.go
       - 整个后端应用的入口点。它负责启动服务器，加载配置，初始化各种组件（如数据库连接、路由等），并设置服务器监听的端口。通过调用其他模块中的函数和方法，将整个后端服务整合起来并运行。
     - config/（配置文件目录）
       - config.go
         - 主要用于读取和管理应用程序的配置。这可能包括数据库连接信息（如数据库类型、主机地址、端口、用户名和密码）、服务器监听端口、日志级别设置等。它会从配置文件（如.env 文件或专门的配置文件格式）中读取这些信息，并以一种方便其他模块访问的方式进行存储和提供。
     - routers/（路由相关）
       - routes.go
         - 定义了后端应用的 API 路由。它将不同的 HTTP 请求方法（如 GET、POST、PUT、DELETE）和路径（如`/users`、`/songs`）与对应的控制器函数进行关联。当接收到一个请求时，路由模块会根据请求的路径和方法找到并调用相应的控制器函数来处理该请求。
     - controllers/（控制器）
       - user_controller.go
         - 处理与用户相关的业务逻辑。例如，用户注册时，它会接收前端传来的用户注册信息（如用户名、密码、邮箱等），调用服务层的用户服务函数来验证和存储用户数据；用户登录时，它会验证用户提供的凭据，生成并返回认证令牌等操作。目前仅实现了注册、登录功能。
       - song_controller.go
         - 负责处理歌曲相关的操作。这包括获取歌曲列表，根据歌曲 ID 获取特定歌曲的详细信息，添加新歌曲、更新歌曲信息（如歌词修改、歌曲分类调整等）以及删除歌曲等功能。它会调用服务层的歌曲服务来执行这些操作，并将结果返回给前端。
       - album_controller.go
         - 处理专辑相关的业务逻辑。例如，获取专辑列表，根据专辑 ID 获取专辑详情（包括专辑封面、专辑中的歌曲列表等），创建新专辑、添加歌曲到专辑、修改专辑信息（如专辑名称、发行日期等）以及删除专辑等操作。它会调用服务层的专辑服务来完成这些操作。
       - playlist_controller.go
         - 管理歌单相关的操作。包括获取歌单列表，根据歌单 ID 获取歌单详情（歌单名称、创建者、包含的歌曲等），创建歌单、向歌单添加歌曲、从歌单删除歌曲、修改歌单信息（如歌单封面、歌单描述等）等功能。它会调用服务层的歌单服务来实现这些操作并返回结果。
       - email_controller.go
         - 目前仅实现了注册时的发送、验证邮件验证码功能。
     - models/（数据模型）
       - user.go
         - 定义用户数据模型。它描述了用户实体在数据库中的存储结构，包括用户的各种属性，如用户 ID、用户名、密码（通常是加密后的）、邮箱、头像 URL、注册时间等。这些模型通常与数据库表结构相对应，用于在代码中方便地操作数据库中的用户数据，如插入、查询、更新和删除用户记录。目前仅实现了插入和查询功能。
       - song.go
         - 定义歌曲数据模型。包含歌曲的基本信息，如歌曲 ID、歌曲名称、歌手、时长、专辑 ID、音乐文件存储路径、歌词等。这个模型用于在后端服务和数据库之间传递和操作歌曲相关的数据，确保数据的一致性和准确性。
       - album.go
         - 定义专辑数据模型。它涵盖了专辑的相关属性，如专辑 ID、专辑名称、歌手、发行日期、封面图片路径、所属音乐风格等。这个模型有助于管理专辑数据，比如在数据库中存储、查询和更新专辑信息。
       - playlist.go
         - 定义歌单数据模型。包括歌单 ID、歌单名称、创建者 ID、创建时间、歌单描述、歌单封面图片路径等信息。它用于处理歌单在数据库中的存储和操作，使得后端可以方便地对歌单进行增删改查等操作。
     - middleware/（中间件）
       - auth.go
         - 鉴权中间件主要用于验证用户的身份。当一个请求需要用户认证才能访问（如获取用户的个人信息、修改用户设置等操作）时，这个中间件会检查请求头或请求参数中是否包含有效的认证令牌（如 JWT - JSON Web Token）。如果令牌有效，它会解析令牌并获取用户信息，将用户信息传递给后续的控制器函数；如果令牌无效，则返回错误信息（如 401 Unauthorized）。
       - logging.go
         - 日志中间件负责记录应用程序的运行日志。它可以记录每个请求的相关信息，如请求的路径、方法、时间、请求来源 IP 地址，以及响应状态码等。通过记录这些日志，可以方便地进行故障排查、性能分析和安全审计等操作。
     - services/（服务层）
       - user_service.go
         - 封装了用户相关的业务逻辑。它会调用数据模型层的函数来与数据库进行交互，实现用户数据的持久化。例如，它包含用户注册的业务逻辑，会对用户输入的密码进行加密处理后存储到数据库；还包括用户登录验证逻辑，通过查询数据库验证用户名和密码是否匹配，并生成认证令牌。此外，还可能包含获取用户信息、更新用户信息等功能。
       - song_service.go
         - 提供歌曲相关的业务服务。它会处理与歌曲数据存储和查询相关的复杂逻辑。例如，在获取歌曲列表时，它可能会根据不同的条件（如按照热度排序、按照发布时间排序等）从数据库中查询歌曲数据；在添加新歌曲时，它会处理歌曲文件的存储路径设置，并将歌曲信息正确地插入到数据库中。
       - playlist_service.go
         - 实现歌单相关的业务操作。它会处理歌单和歌曲之间的关联操作，比如向歌单添加歌曲时，它需要同时更新歌单和歌曲的关联关系在数据库中的记录。它还会处理歌单的创建、查询、修改和删除等操作，确保歌单数据的完整性和一致性。
     - utils/（工具函数）
       - response.go
         - 主要用于格式化后端返回给前端的响应。它可以将数据包装成统一的格式，如包含状态码、消息和数据内容的结构体。这样可以确保前端接收到的响应格式一致，方便前端进行错误处理和数据展示。例如，当请求成功时，返回一个包含状态码 200、成功消息和具体数据的响应；当出现错误时，返回包含相应错误状态码（如 400 Bad Request、404 Not Found 等）和错误消息的响应。
     - database/（数据库相关）
       - db.go
         - 负责建立和管理数据库连接。它会根据配置文件中的数据库信息（如数据库类型是 MySQL、MongoDB 等，主机地址、端口、用户名和密码）来初始化数据库连接。这个文件还可能包含一些基本的数据库操作函数，如获取数据库实例、关闭数据库连接等。
       - redis.go
         - 专门负责存储临时生成的验证码（使用时需要安装redis）
       - migrations/（数据库迁移）
         - init.go
           - 用于初始化数据库迁移。数据库迁移是一种管理数据库结构变化的方式，例如创建新的表、添加或修改表中的列、创建索引等。这个文件可能包含一些函数来执行数据库迁移脚本，确保数据库结构与应用程序的模型定义保持一致，方便在应用程序升级或部署到新环境时更新数据库结构。
   - frontend（前端部分）
     - public/（公共文件）
       - index.html
         - 是整个前端应用的入口 HTML 文件。它包含了页面的基本结构，如`<head>`标签用于设置页面标题、引用 CSS 样式文件和脚本文件；`<body>`标签用于挂载 Vue 应用。当用户访问网站时，浏览器首先加载这个文件，然后通过 JavaScript（特别是 Vue.js）来动态地填充和更新页面内容。
     - src/（源代码）
       - assets/（静态资源）
         - 存放前端应用的各种静态资源，如图片（歌手头像、专辑封面、歌单封面等）、字体文件、CSS 预处理器（如 Sass 或 Less）的源文件等。这些资源可以在组件或页面中通过相对路径引用，用于美化和丰富页面的显示效果。
       - components/（Vue 组件）
         - 包含各种可复用的 Vue 组件。例如，可能有一个通用的歌曲列表组件，它可以接收一个歌曲数据数组作为参数，然后循环展示每首歌曲的名称、歌手、时长等信息；还有播放按钮组件，用于控制歌曲的播放和暂停；评论输入框组件，用于用户输入评论等。这些组件可以在不同的页面视图中重复使用，提高代码的复用性和可维护性。
       - views/（页面视图）
         - Home.vue
           - 代表首页的视图组件。它负责展示首页的内容，如热门歌曲推荐（可能通过调用后端 API 获取数据并展示）、热门歌单推荐、一些广告或活动信息等。这个组件可能会包含多个子组件，如歌曲推荐组件、歌单推荐组件等，以构建完整的首页布局。
         - Rank.vue
           - 排行榜页面的视图组件。它会从后端获取各种排行榜数据（如热歌榜、新歌榜等），并以合适的方式展示排行榜信息，如排名、歌曲名称、歌手、热度指数等。这个组件可能还会提供一些交互功能，如查看歌曲详情、播放歌曲等。
         - Messages.vue
           - 消息中心页面的视图组件。它用于展示用户收到的各种消息，如系统消息（如版本更新通知、活动提醒等）、用户互动消息（如其他用户对自己评论的回复、关注提醒等）。这个组件可能会有消息分类、消息详情查看、删除消息等功能。
         - Search.vue
           - 搜索结果页面的视图组件。当用户在搜索框中输入关键词并进行搜索后，这个组件会接收后端返回的搜索结果数据（可能包括歌曲、歌手、专辑、歌单等），并将这些结果以列表或卡片的形式展示给用户。它还可能提供进一步筛选搜索结果的功能，如按照类型（歌曲、歌手等）筛选。
         - Playlist.vue
           - 推荐歌单页面的视图组件。它会获取后端提供的推荐歌单数据，展示歌单的封面、名称、创建者、简介等信息。用户可以点击歌单查看歌单详情，这个组件可能会通过路由导航到歌单详情页面（PlaylistDetail.vue）。
         - PlaylistDetail.vue
           - 歌单详情页面的视图组件。它展示特定歌单的详细信息，包括歌单中的所有歌曲列表（可以播放歌曲、查看歌曲详情等）、歌单的创建者信息、歌单的描述、歌单的标签等。这个组件可能还会提供一些操作歌单的功能，如收藏歌单、分享歌单等。
         - Player.vue
           - 歌曲播放器页面的视图组件。它负责音乐播放的控制和展示相关信息，如当前播放歌曲的封面、名称、歌手、歌词显示、播放进度条、音量控制等。同时，这个组件可能还会包含评论区，用于展示和添加歌曲评论，评论数据可以通过调用后端 API 进行获取和提交。
         - Profile.vue
           - 我的主页视图组件。它展示用户的个人信息，如头像、用户名、关注数、粉丝数等。还会展示用户的动态（如用户创建的歌单、收藏的歌曲等），这个组件可以通过调用后端 API 获取用户相关数据来填充页面内容。
         - MyMusic.vue
           - 我的音乐页面的视图组件。它会展示用户自己的音乐相关内容，如我的歌手（用户关注的歌手）、我的视频（如果有音乐相关视频功能）、我的电台（如果有电台功能）、用户自己创建的歌单等。这个组件可以通过调用后端 API 获取用户相关的音乐数据来展示。
         - Settings.vue
           - 设置页面的视图组件。它提供各种设置选项，如基本设置（如语言选择、主题切换等）、绑定设置（如绑定手机号、第三方账号绑定等）、隐私设置（如设置个人信息的可见性、评论权限等）。用户在这个页面进行的设置操作会通过调用后端 API 将设置数据保存到服务器。
       - router/（Vue 路由配置）
         - index.js
           - 定义了前端应用的路由规则。它将不同的页面路径（如`/`对应首页、`/rank`对应排行榜页面、`/messages`对应消息中心页面等）与相应的视图组件进行关联。当用户访问某个路径时，路由系统会根据这个文件中的配置加载并显示对应的组件。同时，它还可以设置路由的嵌套关系、路由参数（如通过`/playlist/:id`传递歌单 ID 参数）等，用于构建复杂的页面导航结构。
       - store/（Vuex 状态管理）
         - index.js
           - 是 Vuex 状态管理的核心文件。它用于创建和配置 Vuex 存储，包括定义全局状态对象（如用户登录状态、当前播放歌曲信息、播放列表等）、定义修改状态的 mutations（同步操作）和 actions（可以包含异步操作，如调用后端 API 来更新状态）。通过 Vuex，不同的组件可以方便地共享和修改这些全局状态，保持数据的一致性和响应的同步性。
       - App.vue
         - 是整个 Vue 应用的主组件。它通常包含页面的公共部分，如导航栏（可以在不同页面之间切换）和页脚（可能包含版权信息、联系我们等）。这个组件会包裹其他页面视图组件，通过路由系统来动态显示不同的页面内容。
       - main.js
         - 是前端应用的入口 JavaScript 文件。它主要用于创建 Vue 应用实例，配置应用的相关选项，如挂载点（将 Vue 应用挂载到`index.html`中的某个 DOM 元素上）、引入路由、引入 Vuex 存储、注册全局组件等操作。它是整个前端应用启动的关键文件，将各个模块组合在一起并启动应用。
     - package.json
       - 记录了前端项目的依赖信息。包括项目所使用的各种 JavaScript 库和工具（如 Vue.js、Axios 用于 HTTP 请求、Vue Router 用于路由管理、Vuex 用于状态管理等）的名称、版本号以及其他一些项目相关的脚本命令（如启动开发服务器的命令、构建生产环境代码的命令等）。当在一个新的环境中部署项目时，可以根据这个文件来安装所有必需的依赖项。
